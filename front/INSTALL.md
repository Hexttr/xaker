# Инструкция по установке Frontend

## Быстрый старт

### 1. Установка зависимостей

```bash
cd front
npm install
```

### 2. Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

### 3. Сборка для production

```bash
npm run build
```

Собранные файлы будут в папке `dist/`

## Настройка для нового сервера

### Вариант 1: Разработка (локально)

1. Установите зависимости: `npm install`
2. Запустите dev-сервер: `npm run dev`
3. Убедитесь, что backend запущен на `http://localhost:3000`

### Вариант 2: Production (на сервере)

1. Установите зависимости: `npm install`
2. Соберите проект: `npm run build`
3. Скопируйте содержимое `dist/` на веб-сервер
4. Настройте веб-сервер для обслуживания статических файлов

## Переменные окружения

По умолчанию фронтенд не требует переменных окружения - все настройки в коде.

Если нужно изменить API endpoint, отредактируйте `src/services/api.ts`:

```typescript
const api = axios.create({
  baseURL: '/api',  // Измените на нужный URL
  timeout: 30000,
});
```

## Требования

- Node.js 18 или выше
- npm 9 или выше

## Проверка версии Node.js

```bash
node --version
```

Должно быть: `v18.x.x` или выше

## Устранение проблем

### Ошибка при установке зависимостей

```bash
# Очистите кэш npm
npm cache clean --force

# Удалите node_modules и package-lock.json
rm -rf node_modules package-lock.json

# Установите заново
npm install
```

### Ошибка при сборке

```bash
# Проверьте версию Node.js
node --version

# Если версия старая, обновите Node.js
# Затем повторите установку зависимостей
```

### Проблемы с TypeScript

```bash
# Проверьте типы
npm run type-check
```

## Структура после сборки

После `npm run build` создается папка `dist/` со следующей структурой:

```
dist/
├── index.html
├── assets/
│   ├── index-[hash].js
│   ├── index-[hash].css
│   └── ...
└── favicon.svg
```

Эти файлы нужно скопировать на веб-сервер.
