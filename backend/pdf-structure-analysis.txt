  ШАГ 4: ОБХОД CAPTCHA (AUTH-VULN-05) — КРИТИЧЕСКАЯ   
Описание: 
Атакующий обнаруживает, что Cloudflare Turnstile CAPTCHA на форме обратной 
связи — чисто декоративный элемент. Токен никогда не проверяется сервером. 
Технические детали: 
Фронтенд генерирует токен длиной 1029 символов, но он НИКОГДА не 
отправляется в POST-запросе к /admin/site/feedback/. Прямые API-вызовы без 
CAPTCHA проходят со 100% успешностью. 
Команды/Запросы: 
# Прямой POST без CAPTCHA токена 
curl -X POST https://api-my.tcell.tj/admin/site/feedback/ \ 
  -H 'Content-Type: application/json' \ 
  -d '{"full_name":"Test","phone_num":"+992900000001", 
       "email":"test@test.com","company_name":"Test", 
       "type":"lead","lang_id":1}' 
# Ответ: 201 Created (успешно без CAPTCHA) 
Воздействие: 
CAPTCHA обеспечивает 0% защиты от ботов. Атакующий может 
автоматизировать отправку заявок без каких-либо ограничений.

  ШАГ 6: SSL STRIPPING АТАКА (AUTH-VULN-01 + AUTH-VULN-02)   
Описание: 
При наличии позиции Man-in-the-Middle (например, в публичной WiFi-сети) 
атакующий может провести SSL stripping атаку. 
Технические детали: 
Отсутствие HSTS заголовка позволяет перехватить первое HTTP-соединение до 
редиректа на HTTPS. Cookie NEXT_LOCALE передаётся без флага Secure, что 
делает его уязвимым к перехвату. 
Команды/Запросы: 
# Использование sslstrip или bettercap 
sslstrip -l 8080 
# Или 
bettercap -iface eth0 -caplet hstshijack/hstshijack 
 
# Перехваченные данные: 
# Cookie: NEXT_LOCALE=ru 
Воздействие: 
Перехват cookie и возможность инъекции вредоносного контента в HTTP-трафик. 
Требует физической близости к жертве (публичный WiFi, корпоративная сеть).

5. РЕКОМЕНДАЦИИ ПО УСТРАНЕНИЮ 
5.1 Немедленные действия (P0 — 1-2 дня) 
КРИТИЧНО: Валидация CAPTCHA 
Добавить серверную проверку Cloudflare Turnstile токена. Без этого исправления 
система лидогенерации остаётся полностью беззащитной. 
 
# Django backend 
import requests 
 
def validate_turnstile(token, ip): 
    resp = requests.post( 
        'https://challenges.cloudflare.com/turnstile/v0/siteverify', 
        data={'secret': 'SECRET_KEY', 'response': token, 'remoteip': ip} 
    ) 
    return resp.json().get('success', False)

6. ЗАКЛЮЧЕНИЕ 
Тестирование на проникновение выявило критические уязвимости в области 
защиты от автоматизации, которые могут быть эксплуатированы без специальных 
технических навыков. Основные проблемы сосредоточены в области валидации 
CAPTCHA и rate limiting. 
 
6.1 Сильные стороны 
• Отличная защита от XSS (WAF + React автоматическое экранирование) 
• Отсутствие уязвимостей SSRF (нет пользовательского ввода URL) 
• Корректная защита административных путей (/admin/, /admin/login/) 
• Правильная фильтрация данных на уровне приложения 
 
6.2 Критические слабости 
• CAPTCHA не проверяется сервером (0% защиты от ботов) 
• Полное отсутствие rate limiting на API и формах 
• Публичный доступ к конфиденциальным бизнес-данным 
• Отсутствие HSTS заголовка (уязвимость к SSL stripping)
